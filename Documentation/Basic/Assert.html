Assert Macro

Definition

Assert(condition)

Input parameters

condition
Any expression that produces a boolean result. If condition is false then the execution stops.

Description

The propose of this macro is to detect invalid conditions on the logic of a program. If condition is false then a message is sent to stderr on POSIX systems, or to the debug output on Windows systems. The message indicates the condition, the source file, line and function where the Assert is located. If the program is being run inside a debugger then the execution stops and the control is returned to the debugger. If the program runs outside of a debugger then the program is ended.

On Windows systems GUI programs does not have stderr, in these cases the debug output can be seen using the program DebugView which is part of SysInternals or if the program is being debugged within Visual Studio then the debug output is seen in the Output panel.

This is a macro and does not belong to the Grok namespace.

This macro can be disabled if the NDEBUG macro is defined in the compiler command line or in the project properties under preprocessor definitions.

Example

In this code 'Assert' is used to prevent division by zero. If 'b' is different to zero then the program continues its execution.

float a, b, c;
Assert(b!=0)
c = a/b;

Only if 'b' equals to zero then the execution will stop and an error message like this is sent.

Assert fail: b!=0, source: Test\main.cpp", line: 24, function: main

Requirements

#include <Basic/Assert.h>
